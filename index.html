Entendendo Comunicação Client X Server

↑ Voltar ao topo

A comunicação Client X Server é a estrutura de comunicação que não dependerá de que todas funcionalidades de um determinado programa executem somente na máquina do usuário.

A separação entre a execução de funcionalidades e requerimentos é feita em ambientes distintos.

Esse tipo de sistema independe da comunicação via internet, e pode ser montado em comunicações internas, entre máquinas em uma rede LAN.

História da Web

↑ Voltar ao topo

A história da Web, apesar de profundamente atrelada com a história dos computadores, teve inicio com a ARPANET em 1969. Em meio as inovações criadas no período da Guerra Fria.

Como diversas invenções deste período, a ARPANET tinha seu foco a utilização militar para transmissão de dados sigilosos e interligar instituos de pesquisa nos E.U.A

Posteriormente a isso, a partir de 1984, Tim Berners-Lee, criou o protocolo de transferência de hipertexto HTTP e a liguagem de marcação de hipertextos HTML.

Levando essa tecnologia ao uso popular.

Em 1989 Tim Berners-Lee, uniu o hipertexto com a internet e criou a World Wide Web, o primeiro site criado foi ao ar em 1991 com informações sobre o que é a WWW e sobre os navegadores, o site pode ainda ser acessado. World Wide Web

“A web não está concluída, é apenas a ponta do iceberg. As novas mudanças irão balançar o mundo ainda mais.”Tim Berners-Lee

Clients

↑ Voltar ao topo

Clients são as aplicações e programas que estão do lado do usuário.

Nesse sistema de comunicação, Clients são responsáveis por requisitar as informações dos servers e a partir dos clientes são enviados inputs se existirem.

Normalmente nos Clients são armazenados o mínimo de informação possível para o funcionamento de uma determina aplicação, a maior parcela de informações normalmente estará no lado do server.

Servers

↑ Voltar ao topo

Servers são as máquinas que irão ter as informações sobre a aplicação e irão rodar essas informações para disponibilizá-las aos clients.

Os servers de uma determinada aplicação podem ter diversas estruturas e funcionalidades.

Servers diferentes podem ser designados para realizar tarefas distintas de uma mesma aplicação. Como por exemplo armazenar dados, caixa de e-mail e aplicação web.

Linguagens de Programação

↑ Voltar ao topo

Linguagens de programação são linguagens criadas para comunicar com o computador para executar funções específicas.

Existem diversas linguagens em diversos níveis.

Elas podem estar mais próximas a comunicação das placas lógicas dos computadores, em binário, ou terem um nível maior, e estarem mais próximas da forma de escrita humana.

E existem linguagens que irão atuar na comunicação Cliente x Server em partes diferentes da comunicação, alguns irão atuar no Client Side enquanto outras irão atuar no Server Side.

Exemplos de linguagem de programação Cliente Side: JavaScript.

Exemplos de linguagens de programação Server Side: JavaScript, PHP, C#, Java e Ruby.

HTML é uma linguagem de marcação, não de programação.

Introdução ao HTML na Prática

↑ Voltar ao topo

O HTML por ser uma linguagem de marcação, terá como função na aplicação Web estruturar, o corpo da aplicação e demarcar o esqueleto da aplicação.

A partir desta estrutura básica, é possível utilizar o CSS e JavaScript para gerar estilos e funcionalidade para a aplicação.

Por ser uma linguagem para marcação e não para criar funções, o HTML não é considerado uma linguagem de programação.

Ferramentas Utilizadas

↑ Voltar ao topo

Para a construção de uma página web e estruturar um código HTML, é necessário somente algum programa de texto e que o arquivo criado seja salvo na extensão correta, no caso .html

Porém o recomendado é o uso de um compilador que são programas específicos para a escrita de códigos em linguagens de programação e no caso do HTML, marcação.

Compiladores terão a função de entender o código escrito e no momento da produção e execução testar para erros de escrita ou de lógica, que poderão gerer bugs na execução.

Compiladores auxiliam imensamente na produção de um código.

Um dos compiladores mais famosos e utilizados é o VS Code.

Estrutura Básica do HTML

↑ Voltar ao topo

O HTML é dividido em tags e atributos.

As tags estruturarão o conteúdo do site. Atribuindo padrões e funções ao conteúdo.

Os atributos estarão dentro das tags, atribuindo valores e especificidades as tags.

Tags

↑ Voltar ao topo

Tags são responsáveis por montar a estrutura do código HTML.

As diversas tags terão funções distintas e diversas aplicações estruturais no HTML, como em ordens de importância, alterações de forma de exibição e agregar arquivos ou funções externas no site.

As tags são escritas no HTML entre < > e terão funções específicas, alguns exemplos são:

  • h1 até h6 - Indicam títulos

  • div - Separa o conteúdo interno em um grupo

  • p - Registra um parágrafo

  • ol e ul - Listas ordenadas e não ordenadas respectivamente

  • a - Ancoragem do site, comumente utilizada para navegação

Atributos

↑ Voltar ao topo

Atributos estão integrados as tags de forma a gerar funcionalidades necessárias ou ordenação as tags.

Atributos podem ser gerais e empregrados em qualquer tipo de tag, mas também existem atributos específicos de algumas tags, alguns exemplos são:

  • id - Identificador da Tag, pode ser utilizado em outras partes do código ou no CSS e JavaScript paratratar específicamente da tag com o id

  • class - Similar ao Identificador, porém pode ser utilizado em diversas tags para serem atribuidos estilos ou funções similares a várias partes da estrutura.

  • href - Atrelado a tag a, a função deste atributo é referenciar onde é a ancoragem da tag, seja um link externo ou interno.

  • target - Pertence a tag a, terá a função de indicar ao navegador se o link irá ser aberto na mesma página ou em um nova aba.

  • src - Atrelado a tag img, indica o caminho onde será buscada a imagem, seja nos arquivos da aplicação ou externamente.
